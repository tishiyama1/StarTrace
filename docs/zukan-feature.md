# 図鑑機能(ほしぞら ずかん)設計書

## 1. 目的

なぞって見つけた星座を集めていく「コレクション体験」を加え、子供が繰り返し遊びたくなる
動機付けをつくる。見つけた星座は端末に保存され、いつでも一覧で振り返れる。

## 2. 発見(ディスカバリー)の判定

- なぞりの判定結果(`MatchResult`)の `score`(0〜100 のマッチ度)が
  **発見閾値以上**のときに、その星座を「発見済み」として登録する。
- 閾値は `src/lib/shapeMatcher.ts` の `DISCOVERY_SCORE_THRESHOLD`(既定 `65`)。
  `NOT_FOUND_SCORE_THRESHOLD` と同じ値で、「結果が表示された = 図鑑にも登録される」
  という子供に分かりやすい体験にしている。
  - 値はスコア分布の計測で決定: ランダムななぐり書きは中央値59%・最大79%、
    手ブレを加えた正しいなぞりは最低でも88%。65%はなぐり書きの約9割をブロック
    しつつ、正しいなぞりを誤ってブロックしない。
- 閾値未満のときは星座名を出さず「うーん、みつからないね…」の演出を表示し、
  再挑戦と図鑑のヒントへ誘導する(`NotFoundOverlay`)。

## 3. 永続化

- `src/hooks/useDiscoveries.ts` が発見済み星座IDの集合を管理する。
- 保存先は `localStorage`、キーは `startrace.discoveries.v1`(値は ID 文字列の配列 JSON)。
  - バージョン付きキーにしてあるので、将来データ形式を変えるときに移行しやすい。
- `localStorage` が使えない/壊れている場合(プライベートモード等)は例外を握りつぶし、
  「空の図鑑・保存はされない」状態で問題なく遊べるようにフォールバックする。
- `add(id)` は登録と同時に「今回はじめての発見か(=新発見か)」を **同期的に** 返す。
  React の state 更新は非同期のため、最新状態を `useRef` にミラーして判定している。

## 4. UI

### 4.1 図鑑を開くボタン

- 画面右上に常時表示。現在の発見数を `発見数/全数`(例: `3/22`)で示す。

### 4.2 結果画面の新発見演出

- なぞった直後の結果カードで、今回はじめて発見した星座には
  「✨ ずかんに とうろく! ✨」のリボンとポップ演出(`result-panel--new`)を出す。
- あわせてカテゴリ(実在/おはなし)を色分けバッジで表示する。

### 4.3 図鑑画面(`Zukan.tsx`)

- **進捗**: 「◯ / ◯ こ はっけん!」の見出しとプログレスバー。
- **絞り込み**: 「ぜんぶ / ほんとうの せいざ / おはなしの せいざ」のチップで切り替え。
- **カード グリッド**:
  - 発見済み: 絵文字 + お手本の形(`ConstellationDiagram`)+ 名前 + 紹介文。
  - 未発見: 「?」のミステリーカードで形と名前を隠し、「なぞって みつけよう」と促す。
- **リセット**: 図鑑を空に戻せる。誤操作防止のため「ぜんぶ けしても いい?」の確認を挟む。

## 5. データ駆動

図鑑の中身はすべて `CONSTELLATIONS` データから生成される。星座を追加すれば、
判定・図鑑・進捗の分母・カテゴリ絞り込みに自動で反映される(コード変更不要)。

## 6. テスト方針

- 発見・永続化ロジック(`useDiscoveries`)は React フックのため、E2E(Playwright)で
  「新発見 → 図鑑に反映 → 進捗が増える」動作を確認する。
- 発見閾値の定数は `shapeMatcher` に置き、判定ロジックの単体テストと同じ場所で管理する。
