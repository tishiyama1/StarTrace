# 星座テンプレートデータ仕様書

## 1. 目的

なぞり判定の比較対象となる「お手本の形」を定義する。実際の星の正確な赤経・赤緯ではなく、
**子供が見て「その星座らしい」と感じられる、特徴を誇張した簡略形状** として座標を定義する
(スマホの小さい画面で指でなぞって認識できる程度にシンプル化している)。

## 2. データ構造

`src/data/constellations.ts` に以下の型で12星座分を定義する。

```ts
export interface ConstellationPoint {
  x: number;
  y: number;
}

export interface Constellation {
  id: string;          // 一意なID (例: "ursa-major")
  nameJa: string;       // 表示名 (例: "おおぐま座")
  nameEn: string;       // 通称英語名 (例: "Big Dipper (Ursa Major)")
  latinName: string;    // 学名 (例: "Ursa Major")
  path: ConstellationPoint[]; // なぞる順に星を直線で結んだ座標列(1ストローク分)
}
```

- `path` は「星をこの順番で直線に結ぶとその星座の形になる」という頂点列。
- 座標系は任意単位の 0〜100 四方(相対値)。マッチング時に平行移動・拡大縮小・回転を
  正規化して比較するため、絶対座標やスケールに意味はない。
- 同じ点に戻る(例: 十字形やひし形を1ストロークで描く)場合は `path` 内に同じ座標を
  複数回含めてよい。

## 3. 収録する12星座と形状の設計方針

| # | id | 表示名 | 形の特徴 | 頂点数 |
|---|---|---|---|---|
| 1 | `ursa-major` | おおぐま座(北斗七星) | ひしゃく(柄杓)型のジグザグ | 7 |
| 2 | `ursa-minor` | こぐま座 | 小さいひしゃく型 | 7 |
| 3 | `cassiopeia` | カシオペヤ座 | アルファベットの「W」 | 5 |
| 4 | `orion` | オリオン座 | 砂時計(ちょうちょ結び)型 | 6 |
| 5 | `scorpius` | さそり座 | 先端が巻き上がる「J」字カーブ | 8 |
| 6 | `cygnus` | はくちょう座(北十字) | 十字(プラス記号) | 7(中心を3回通過) |
| 7 | `lyra` | こと座 | 小さなひし形(たこ型) | 5(始点に戻る) |
| 8 | `aquila` | わし座 | 横長のひし形(たこ型) | 5(始点に戻る) |
| 9 | `gemini` | ふたご座 | 頭が寄り添い足が開く2本足型 | 6 |
| 10 | `taurus` | おうし座 | 角が伸びた浅い「V」字 | 5 |
| 11 | `leo` | しし座 | 鎌(?マーク)+三角のしっぽ | 7 |
| 12 | `pegasus` | ペガスス座(秋の四辺形) | 正方形 | 5(始点に戻る) |

実際の星座絵・星の並びを参考にしつつ、上記のように**一筆書きでなぞれる代表的なシルエット**に
単純化している。天文学的な正確さより「子供が指でなぞって再現しやすく、他の星座と形が
被らないこと」を優先している。

## 4. データの妥当性チェック

- 各星座の `path` は最低3頂点以上を持ち、直線が完全に重ならない(=面積・形状として
  縮退しない)こと。
- 12星座それぞれの形が視覚的に大きく異なること(ユニットテストで形状間の距離が
  一定以上離れていることを確認する `constellations.test.ts` を用意する)。
