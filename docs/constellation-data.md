# 星座テンプレートデータ仕様書

## 1. 目的

なぞり判定の比較対象となる「お手本の形」を定義する。

- **実在の星座(`real`)**: 主要星の実際の位置(J2000 の赤経・赤緯)を平面投影して
  0〜100 座標に正規化したもの。実際の夜空・星図で見える形とほぼ同じプロポーション
  で、なぞる順序も一般的な星のつなぎ方(例: 北斗七星はドゥーベ→…→アルカイド)に
  合わせている。頂点は主要星のみに絞り、1ストロークでなぞれるようにしている。
- **おはなしの星座(`fun`)**: アプリオリジナルの形。子供が喜ぶモチーフを
  一筆書きできるシルエットとしてデザインしたもの。

## 2. データ構造

`src/data/constellations.ts` に以下の型で全22星座を定義する。

```ts
export type ConstellationCategory = 'real' | 'fun';

export interface Constellation {
  id: string;                   // 一意なID (例: "ursa-major")
  nameJa: string;               // 表示名 (例: "おおぐま座")
  nameEn: string;               // 通称英語名 (例: "Big Dipper (Ursa Major)")
  latinName: string;            // 学名 (例: "Ursa Major")
  category: ConstellationCategory; // 'real'=実在 / 'fun'=おはなしの星座(非実在)
  emoji: string;                // 図鑑カード等に添える絵文字
  hint: string;                 // 未発見の図鑑カードに出す「なぞる形」のヒント
  description: string;          // 結果画面・図鑑に表示する子供向けの紹介文
  path: Point[];                // なぞる順に星を直線で結んだ座標列(1ストローク分)
}
```

- `path` は「星をこの順番で直線に結ぶとその星座の形になる」という頂点列。
- 座標系は任意単位の 0〜100 四方(相対値)。マッチング時に平行移動・拡大縮小・回転を
  正規化して比較するため、絶対座標やスケールに意味はない。
- 同じ点に戻る(例: 十字形やひし形を1ストロークで描く)場合は `path` 内に同じ座標を
  複数回含めてよい。
- `category` は結果画面・図鑑での色分けと絞り込みに使う。`fun` はこのアプリオリジナルの
  非実在の星座(ふざけた星座)で、学名(`latinName`)も遊びで付けたもの。

実際の星座は星の並びを参考にしつつ、**一筆書きでなぞれる代表的なシルエット**に単純化している。
「おはなしの星座」は完全にオリジナルで、子供が喜ぶモチーフを一筆書きできる形にした。
いずれも天文学的な正確さより「子供が指でなぞって再現しやすく、他の星座と形が被らないこと」を優先している。

## 4. データの妥当性チェック(単体テスト)

`src/lib/shapeMatcher.test.ts` で全 `CONSTELLATIONS` をループして検証する。

- 各星座を「平行移動 / 拡大縮小 / 回転 / 向き反転」した入力が、いずれも自分自身に最も
  高いマッチ度で一致すること。
- **識別性**: どの星座も、全テンプレートの中で自分自身との距離が最小になること
  (=どの2つの形も十分に離れていること)。星座を追加するときは、このテストが通ることで
  既存の形と衝突しないことを保証する。
