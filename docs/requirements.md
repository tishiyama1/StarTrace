# StarTrace 要求仕様書

## 1. 概要

**StarTrace** は、子供向けの星座あてWebアプリです。
画面いっぱいに広がる星空を指(またはマウス)でなぞると、なぞった軌跡の「形」に最も近い星座を判定して表示します。

- タイトル案: StarTrace(ほしのなぞりがき)
- 対象: 未就学児〜小学校低学年とその保護者
- 想定デバイス: スマートフォン・タブレットのブラウザ、PCブラウザ(タッチ/マウス両対応)

## 2. 決定事項(ヒアリング結果)

| 項目 | 決定内容 |
|---|---|
| プラットフォーム | Webアプリ(ブラウザ)。インストール不要ですぐ遊べることを優先。 |
| 対象星座 | 有名な星座 10〜15個程度に絞る(88星座すべては対象外)。 |
| 判定方式 | 形状マッチング。なぞった軌跡を正規化し、各星座の形(星を結んだ線)と比較して最も近いものを選ぶ。位置・大きさ・向きが多少違っても許容する。 |
| 追加機能 | 第一段階では実装しない。コア体験(なぞる→判定→結果表示)のみに集中し、演出・豆知識・図鑑機能は将来拡張とする。 |

### 2.1 第二段階での拡張(実装済み)

初期リリース後、以下の拡張を実装した(本ドキュメントの §3.4〜§3.6)。

| 拡張 | 内容 |
|---|---|
| 対象星座の拡充 | 実在の星座を14個に増やし、さらにこのアプリオリジナルの「おはなしの星座」(モナリザ座・ハート座・ネコ座などの非実在のふざけた星座)8個を追加。合計22個。 |
| 図鑑機能 | 見つけた星座を `localStorage` に記録し、一覧・進捗・カテゴリ別に振り返れる「ほしぞら ずかん」を追加。 |
| ビジュアル強化 | 天の川の帯、定期的に流れる流れ星、ぼんやり光る星雲、星の色・明るさの多様化により、よりリアルで魅力的な星空にした。 |

## 3. 機能要件

### 3.1 コア体験(必須)

1. アプリを開くと、装飾用のランダムな星が散りばめられた夜空の画面が表示される。
2. ユーザーは画面上を指(タッチ)またはマウスでなぞる(1ストロークのドラッグ操作)。
3. なぞっている間、軌跡が光る線としてリアルタイムに描画される。
4. 指を離す(ドラッグ終了)と、なぞった軌跡の形状と、あらかじめ用意された星座テンプレート群の形状を比較し、最も近い星座を1つ判定する。
5. 判定結果として、以下を表示する。
   - 星座名(日本語名 + 学名/英語名)
   - マッチ度(何%近いか、またはスター評価)
   - お手本の星座の形(星をつないだ線)を軌跡と重ねて表示
6. 「もう一度なぞる」ボタンで画面をクリアし、再挑戦できる。

### 3.3 UI/UX 要件

- 子供向けであることを踏まえ、文字は大きく・平易な言葉・漢字は最小限(ひらがな中心 + ふりがな相当の表記)にする。
- 操作は「なぞるだけ」のワンアクションにし、複雑なメニューは持たない。
- 誤判定でも否定的な表現をせず、常に前向きなフィードバックを返す(例:「◯◯座に近いね!」)。
- レスポンシブ対応(スマホ縦持ち・横持ち・タブレット・PCで破綻しないレイアウト)。

### 3.4 図鑑機能(ほしぞら ずかん)

1. 画面右上のボタンから図鑑を開ける。ボタンには現在の発見数(例: `3/22`)を表示する。
2. マッチ度が一定以上(既定 65%)のときに、その星座を「発見済み」として `localStorage` に記録する。
   一定未満のときは星座名を出さず「みつからないね」の演出を返す(なぐり書き対策)。
3. なぞった直後の結果画面で、今回はじめて発見した星座には「ずかんに とうろく!」の演出を表示する。
4. 図鑑画面では以下を表示する。
   - 進捗(発見数 / 全星座数)とプログレスバー
   - 「ぜんぶ / ほんとうの せいざ / おはなしの せいざ」でのカテゴリ絞り込み
   - 星座カードのグリッド。発見済みは絵文字・お手本の形・名前・紹介文を表示し、未発見は「？」でシルエットを隠す。
5. 図鑑は「リセット」でき、誤操作を防ぐため確認ステップを挟む。

### 3.5 ビジュアル要件(リアルな星空)

1. 奥から手前へ「背景グラデーション → 星雲 → 天の川 → 星 → 流れ星」の順で重ねて描画する。
2. 星は白・青白・暖色などに色を散らし、明るい星には十字の光条(フレア)を付ける。
3. 天の川はななめの帯として、ぼんやりした光のムラと帯沿いに集まる細かい星で表現する。
4. 星雲は数か所に配置し、ゆっくり明滅させる。
5. 流れ星は数秒おきにランダムに出現し、尾を引いてななめに流れて消える。
6. すべて Canvas 上でリアルタイムに描画し、なぞり操作の描画性能を損なわない。

### 3.6 その他のUX

- 結果画面に星座のカテゴリ(実在/おはなし)を色分けバッジで表示する。
- 星座名の先頭に絵文字を添え、直感的に分かりやすくする。

## 4. 非機能要件

- 追加のバックエンド・データベースは不要(すべてクライアント側で完結する静的Webアプリ)。
- オフラインでの完全動作は必須要件としない(将来PWA化は拡張候補)。
- 対応ブラウザ: 最新の Chrome / Safari / Edge / Firefox(モバイル含む)。
- 判定はクライアントサイドのJavaScript/TypeScriptのみで完結し、外部APIに依存しない。

## 5. スコープ外(現時点では実装しない)

- 効果音・BGM(音声アセットの管理が必要なため見送り)
- 星座の神話・物語の本格的な解説(現状は1〜2文の紹介のみ)
- 88星座すべての対応、季節ごとの星空切り替え
- ユーザーアカウント・サーバー同期(進捗は端末内 `localStorage` のみ)
- PWA化・オフライン完全対応

上記は将来の拡張候補として `docs/architecture.md` に拡張ポイントとして触れる。
図鑑・演出・ビジュアル強化・星座拡充は第二段階で実装済み(§2.1)。
