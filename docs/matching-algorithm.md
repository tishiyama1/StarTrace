# 形状マッチングアルゴリズム設計書

## 1. 目的

ユーザーが指(マウス)でなぞった1本のストローク(座標列)と、全星座それぞれの
お手本形状(`Constellation.path`)を比較し、最も形が近い星座を1つ選び、
どれくらい近いか(マッチ度)を数値化する。星座は実在14 + おはなし8 の計22個
(`docs/constellation-data.md` 参照)で、実在・非実在の区別なく同じアルゴリズムで判定する。

判定は **平行移動・拡大縮小・回転に対して不変** (どこに・どんな大きさで・どんな向きで
描いても、形が同じなら同じ星座と判定される)であることを目標とする。

## 2. 全体の処理フロー

```
matchConstellation(rawStroke, constellations):
  1. 前処理: rawStroke の重複点除去・極端に短い入力の除外
  2. リサンプリング: rawStroke と 各constellation.path を
     弧長ベースで N=64 点に等間隔リサンプリング
  3. 正規化: 重心を原点に平行移動 + RMS距離が1になるようスケーリング
  4. 各星座について、正方向・逆方向(始点/終点を反転)の2パターンで
     最適回転角を閉形式(Procrustes)で求め、点群間の平均距離を計算
  5. 全星座(22個)× 2方向 = 44通りの中から距離が最小の組み合わせを採用
  6. 距離をマッチ度(0〜100%)に変換
  7. { constellation, score, distance } を返す
```

## 3. 各ステップの詳細

### 3.1 前処理

- `pointerdown`〜`pointerup` の間に記録した座標列から、同一座標の連続(指が止まった)を
  間引く。
- 総移動距離(弧長)が閾値未満(例: Canvasの短辺の5%未満)の場合は「タップ」とみなし、
  マッチングを行わずユーザーに「もう少し大きくなぞってね」とフィードバックする。

### 3.2 リサンプリング(`resample.ts`)

- 折れ線の全長 `L` を計算し、`L / (N-1)` 間隔で線分上を等間隔に補間して N=64点を
  生成する( `$1 Unistroke Recognizer` と同様の手法)。
- ストロークとテンプレートの両方を同じ点数 N にリサンプリングすることで、
  頂点数が違っても点同士を1対1で対応させて比較できるようにする。

### 3.3 正規化(`shapeMatcher.ts`)

- **平行移動**: リサンプリング後の点群の重心 `(cx, cy)` を求め、全点から差し引く。
- **スケーリング**: 重心からの距離の二乗平均平方根(RMS)を求め、それが `1` になるよう
  全点をスケーリングする。バウンディングボックスでの正規化ではなく **RMS距離** を
  用いる理由は、直線的な形状(棒状)でもバウンディングボックスの縦横比が不安定に
  ならないようにするため。

### 3.4 回転の最適化(Procrustes回転)

2つの正規化済み点群 `P = {p_i}`(ユーザー)、`Q = {q_i}`(テンプレート、対応する順序)が
与えられたとき、`Q` を角度 `θ` だけ回転して `P` との距離を最小化する `θ` は、
以下の閉形式で一意に求まる(2次元における最小二乗回転フィッティング)。

```
θ = atan2( Σ (p_i.y * q_i.x − p_i.x * q_i.y),
           Σ (p_i.x * q_i.x + p_i.y * q_i.y) )
```

この式により、角度を総当たりで探索することなく最適な回転角を一発で求められる
(ブルートフォースの角度探索より高速かつ厳密)。

回転後、対応点間のユークリッド距離の平均を「距離スコア」とする。

```
distance = (1/N) * Σ | rotate(q_i, θ) − p_i |
```

### 3.5 始点・向きの不変性

- ユーザーはお手本と同じ向き(星座の「始点」から辿るとは限らない)でなぞる可能性が
  高いため、各テンプレートについて **元の順序** と **反転した順序(終点から始点)** の
  両方で距離を計算し、小さい方を採用する。
- 一部の対称的な形状(正方形・ひし形など)は始点をずらして描かれる可能性もあるが、
  今回のスコープでは「向きの反転」のみ対応し、開始点のシフト(巡回シフト)までは
  対応しない(将来拡張候補)。

### 3.6 マッチ度スコアへの変換

RMS距離1に正規化した点群同士の平均距離は、実験的に概ね `0`(完全一致)〜`1.6`程度
(かなり異なる形)の範囲に収まることを確認した上で、以下の式でスコア化する。

```
score = clamp( 100 * (1 − distance / DISTANCE_SCALE), 0, 100 )
```

`DISTANCE_SCALE` は実際に手描きストロークでテストしながら `shapeMatcher.ts` 内で
定数として調整する(初期値は 1.4 を仮定し、単体テストのケースで妥当性を検証する)。

## 4. 精度・性能に関する補足

- N=64点・22星座×2方向=44パターンの比較はいずれも軽量な行列演算のみで、
  Canvas操作中でも指を離した瞬間に同期的に計算して問題ない処理量。
- 完全な鏡映(左右反転)には対応しない。左右反転した形状は「別の見た目」として
  扱い、正しい星座と判定されない場合がある。子供が直感的に描く分には大きな
  問題にならないと判断し、初期スコープでは対応を見送る。

## 5. テスト方針

`src/lib/shapeMatcher.test.ts` で以下を検証する。

1. あるテンプレートを「そのまま」「平行移動して」「拡大縮小して」「回転させて」
   「向きを反転させて」入力した場合、いずれも同じ星座が最高スコアで一致すること。
2. 全く違う形状(例: 直線 vs 正方形)を入力した場合、スコアが低くなること。
3. 全22星座同士の形状距離が十分離れており、どの星座も自分自身との距離が最小になる
   こと(テンプレート同士の識別性チェック)。新しい星座を追加したときも、このテストが
   通ることで既存の形と衝突しないことを保証する。

なお、マッチ度が `NOT_FOUND_SCORE_THRESHOLD`(既定65%)未満のときは、どの星座も
表示せず「みつからないね」の演出を返す(適当ななぐり書き対策。閾値はなぐり書きと
正しいなぞりのスコア分布の計測から決定)。それ以上のときは結果を表示し、同時に
図鑑へ登録する(`DISCOVERY_SCORE_THRESHOLD` = 同値。詳細は `docs/zukan-feature.md`)。
