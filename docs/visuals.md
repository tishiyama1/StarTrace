# 夜空ビジュアル設計書(実写志向レンダラー)

## 1. 目的

「実際の星空写真のように美しく、ずっと眺めていたくなる夜空」を Canvas 上に描く。
天体写真の構成要素(天の川の雲状構造と暗黒帯・銀河バルジ・恒星の色温度・
ブルーム・回折光条・大気光・前景シルエット・周辺減光)を再現しつつ、
子供のなぞり描きが 60fps で快適に動くことを両立する。

## 2. アーキテクチャ: 静的プリレンダ + 動的レイヤー

描画は2つのモジュールに分かれる。

- **`src/lib/skyRenderer.ts`(静的な空)** — コストの高い要素を
  `renderStaticSky()` で初回(とリサイズ時)に一度だけオフスクリーンキャンバスへ
  描く。毎フレームは `drawImage` 1回の転写で済むため、どれだけ密度を上げても
  実行時コストは一定。
- **`src/lib/skyEffects.ts`(動く演出)** — 流れ星・火球・流星群・人工衛星・
  オーロラのイベント(従来どおり)。

### 静的レイヤーの内容(`renderStaticSky`)

1. **空のベース + 天の川(低解像度 per-pixel パス)**: 実寸の約0.3倍の
   ImageData に対して1ピクセルずつ計算し、拡大転写でなめらかににじませる。
   - 縦グラデーション(深い藍 → 紫紺)
   - **天の川**: 画面対角に沿う帯。fBm(フラクタルノイズ)による雲状のムラ、
     中心を蛇行する**暗黒帯(グレートリフト)**、片側の**銀河バルジ**
     (暖色のふくらみ)で構成。外側は青白く、バルジは象牙色。
   - **星雲**: 3か所のごく淡い色のにじみ(ピンク・青・紫)。ノイズで形を崩す。
   - **大気光(エアグロー)**: 地平線近くのかすかな緑の帯。
   - **周辺減光(ビネット)**: 四隅をわずかに落とし、写真らしい落ち着きを出す。
2. **微光星 約2,000個**(実解像度): 小さいほど多い等級分布。45%は天の川の帯に
   ガウス分布で密集させ、帯の「星の雲」感を作る。
3. ノイズと配置は**固定シード**で決定的 — いつ開いても「同じ夜空」になる。

### 動的レイヤー(毎フレーム)

1. 静的な空の転写(`drawImage` 1回)
2. オーロラ(イベント時。星はオーロラ越しに透ける)
3. **輝星 約130個**: 色温度パレット(青白 O/B → 白 → 黄 → 橙 K/M)から重み付き
   抽選。ガウス的なにじみ(PSF)を焼き込んだ**スプライト**を使い回して描画し、
   ひときわ明るい星には**十字の回折光条**が付く。またたきは低速の明滅+
   速い揺らぎ(大気のシンチレーション)の2重サイン波。
4. 流れ星・火球・人工衛星(イベント)
5. **山影シルエット**(前景): fBmの稜線ともみの木。流れ星は山のうしろに沈む。
6. お手本ライン / なぞりの光る線(最前面)

実測パフォーマンス: スマホ相当(390×844 @2x)で **60fps**、初期化 約0.3秒。

## 3. 座標系とリサイズ

- 天体の座標はすべて **0〜1 の正規化値**で保持し、描画時に `width`/`height` を掛ける。
  これによりウィンドウのリサイズや端末の違いに依存せず、生成し直す必要がない。
- 静的な天体(背景星・天の川の星・星雲)は初回だけ生成して `useRef` に保持する。
- Canvas は `devicePixelRatio` を掛けた実解像度で用意し、`setTransform` で論理座標に戻す
  (高精細ディスプレイでもぼやけない)。

## 3.5 夜空イベント(眺めるだけでも発見がある空)

流れ星が画一的にならないよう、`SkyCanvas` のイベントスケジューラが数秒おきに
次のどれかを起こす。レアなものほど出にくく、「たまたま見られた」体験を作る。

| イベント | 頻度 | 内容 |
|---|---|---|
| ふつうの流れ星 | よく起こる | 色(白/青白/緑/オレンジ/青紫)・太さ・速さ・尾の長さが1本ずつ違う |
| 火球(ファイアボール) | ときどき | 大きく明るい金色の流れ星。頭が強く光り、残光のグローをまとってゆっくり流れる |
| 流星群 | ときどき | 数秒のあいだに4〜7本が立てつづけに流れる |
| 人工衛星 | ときどき | またたかない小さな点が、流れ星の1/10ほどの速さで、すーっと等速で横切る |
| オーロラ | いちばん珍しい | 緑〜紫の2〜3層のカーテンが画面上部で15〜25秒ゆらめき、ゆっくり現れて消える |

- 流れ星の色は実際の流星と同じ理屈(組成による発光色の違い)で、白〜青白を
  基本に緑(マグネシウム)・オレンジ(ナトリウム)などを混ぜている。
- オーロラは波打つ上端ライン(2つのsin波の重ね合わせ)+縦方向グラデーションの
  カーテンを層ごとに描画し、寿命に沿ってフェードイン/アウトする。
- 検証用の隠しフック: URL に `?sky=aurora` / `shower` / `fireball` / `satellite` を
  付けると、そのイベントがすぐに発生する(通常利用では無害)。

## 4. 流れ星の挙動

- **出現**: `nextSpawn` タイマーが 0 を切ると1つ生成し、次の出現までを 3.5〜8.5 秒の
  ランダムに設定する。同時に存在するのは最大2個まで。
- **進行方向と出現位置**: 画面上部から下向き・ななめに流れる。進行方向と **反対側**から
  出発させることで(右へ流れるものは左寄りから)、画面をしっかり横切って見えるようにする。
- **速度・寿命**: 約 1.1〜1.6 秒かけて画面を横切る速さに調整。速すぎると一瞬で消えて
  気づけないため、あえてゆっくりめにしている。
- **見た目**: 進行方向へ伸びる尾を、後端が透明・先端が明るいグラデーションの線で描き、
  先端に小さな輝き(グロー)を足す。出現直後にすっと現れ、消える直前にフェードアウトする。
- 位置更新・寿命/画面外判定は副作用のない関数 `updateShootingStars()` に切り出し、
  単体テスト(`skyEffects.test.ts`)で「前進する / 寿命切れで消える / 画面外で消える」を検証する。

## 5. パフォーマンス配慮

- 背景星 約170個、天の川の星 約230個、星雲3個、流れ星 最大2個と、いずれも軽量な図形描画。
- 流れ星以外は状態を持たないため、毎フレームの計算は最小限。なぞり操作中でも
  60fps を維持でき、入力描画の応答性を損なわない。
- フレーム時間 `dt` は上限(64ms)でクランプし、タブ非アクティブ復帰時などに
  流れ星がワープしないようにしている。

## 6. 今後の拡張余地

- `prefers-reduced-motion` に応じて流れ星・明滅を抑えるモード。
- 時間帯・季節による色調変化、月の満ち欠けなどの追加天体。
