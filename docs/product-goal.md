# プロダクトの目的(North Star)

改善ループ(Builder / Reviewer)が**目的に向けて自律的に判断する**ための、
目的(North Star)と、その測り方・守るべき制約を定義する。

> これは PoC。目的そのものより「**目的に対して自律的に提案・改善・検証できるか**」の
> 検証が狙い。獲得(マーケ)やコード外の要素には踏み込まない。収益化は本アプリでは
> 目的にしない(子ども向け・匿名アプリのため、法・倫理的に不適切)。

## North Star(目的) — 単一指標ではなく「体験の質」のバスケット

**「子どもが、なぞって"正しく"星座を見つけられ、集めて楽しい」体験の質を高める。**

一点の数値ではなく、次の3軸のバスケットで捉える。**優先度は上から**。

1. **判定の正確さ(主・最重要)** — なぞりが"正しく"解決される質。指標 = **ネット正解率**(下記)。
2. **コンテンツの広がり** — 集められる星座の種類・要望への対応。指標 = 1人あたりユニーク発見数、要望の消化。
3. **楽しさ・体験** — 見た目・手触り・また来たくなる仕掛け(ダークパターンは除く)。指標 = 肯定的FB比率、(将来)再訪率。

## 主指標:ネット正解率(旧「トレース成功率」の置き換え)

- **定義**: `正しく本人の星座にマッチした数 − 誤って別の星座に誤マッチした数`(多いほど良い)。
- **測り方**: 実トラフィックは少なくノイズが大きいので、**`shapeMatcher` のテスト内シミュレーション**
  (各テンプレートに段階的な手描きノイズを与え、正解/誤りを集計する。PR #30・#38 で確立済みの手法)で推定する。
- 実データの **トレース成功率** `trace_hit / (trace_hit + trace_notfound)` は、**参考指標として日次レポートに
  載せ続ける**が、**これ単体を最大化目標にはしない**(判定を甘くすれば上がってしまうため)。

### なぜ「成功率」でなく「ネット正解率」なのか(重要)
「成功率↑ かつ 新規誤マッチ0」という**過剰制約**は、似た星座(gemini/taurus, heart/pegasus 等)では
両立せず**膠着**する(実際に #38 で発生)。ネット正解率なら、
「**正しいなぞりを誤って not-found に倒す過剰ブロックを直す**」ことと「**誤マッチが少し増える**」ことを
**同じ土俵で天秤にかけて良い**。過剰ブロックを大きく減らせるなら、少数の誤マッチ増は**ネットで正解が
増える限り許容**する。

## ⭐ ガードレール(体感品質) — 数値が良くても守る

- ネット正解率が数値上改善しても、**ユーザーが「ちがう星座が出た」と感じる誤マッチ、特に
  "自信満々の誤判定"(確信度高く別物に断定)を大きく増やす変更は不可**。
- 判定を無差別に甘くして数字を稼ぐのは引き続き禁止。
- つまり許容するトレードは「**ネット正解が増え、かつ体感の悪い誤判定を増やさない**」範囲に限る。

## ★ 視野を狭めないルール(重要)

- 主指標(判定の正確さ)を**安全に動かせる施策が無い日**は、ループは**膠着せず**、
  ガードレールを守れる範囲で**バスケットの他の軸(コンテンツ #2 / 楽しさ #3 / 使い勝手)を進めてよい**。
- North Star は「**何を優先するか**」の軸であって、「それ以外を禁止する排他フィルタ」ではない。
  一つの軸が詰まったら、別の価値を作る。

## 運用ルール(Builder / Reviewer への判断軸)

- **Builder(提案・実装)**:
  - 日次レポートに、参考指標(トレース成功率)の現在値・傾向に加え、**主軸(判定の正確さ)が
    今どういう状態か**を書く。
  - 改善提案は上記バスケットの優先度で並べる。主軸が詰まっている日は他の軸の改善を提案してよい。
  - 判定に関わる変更をしたら、翌日以降のデータ + シミュレーションで**効果を答え合わせ**する(事後検証)。
- **Reviewer(承認・マージ)**:
  - 承認の優先順位にバスケットの優先度を反映(判定の正確さ > コンテンツ > 楽しさ)。
    ただし主軸に安全な候補が無ければ、他軸の価値ある改善を承認してよい(膠着させない)。
  - **ガードレール(体感の悪い誤マッチを増やす恐れ)に反する提案・PR は承認/マージしない。**

## やってはいけないこと

- 子どもを不必要に長居・依存させるダークパターン(通知連打、射幸性など)。
- 体感品質(正しさ・気持ちよさ)を損なってまで数値を稼ぐこと。
- 判定を無差別に甘くしてネット正解率でなく見かけの成功率だけ上げること。
- プライバシー方針(匿名・個人情報を集めない)の変更。
